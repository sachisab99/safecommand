## What was done

### 1. Root-crashed Railway api service — diagnosed and fixed

**Root cause:** All 4 internal packages (`@safecommand/types`, `@safecommand/schemas`, `@safecommand/db`, `@safecommand/queue`) had `exports` fields with only `"import"` condition. TypeScript with `NodeNext` module resolution compiles to CommonJS, so Node.js `require()` calls hit `ERR_PACKAGE_PATH_NOT_EXPORTED` at process startup before the server could bind to any port. Railway healthcheck saw "service unavailable" (connection refused) within 30s and killed the replica.

**Fix:** Added `"require"` and `"default"` conditions to all 4 package `exports` fields:
```json
"exports": {
  ".": {
    "types": "./dist/index.d.ts",
    "require": "./dist/index.js",
    "default": "./dist/index.js"
  }
}
```

**Files changed:**
- `packages/types/package.json`
- `packages/schemas/package.json`
- `packages/db/package.json`
- `packages/queue/package.json`

**Verified locally:** `NODE_ENV=production node apps/api/dist/index.js` → server starts → `GET localhost:4000/health` → 200 `{"status":"ok","checks":{"database":"ok"}}`

**Commit:** `6c5b186` — `fix: add require+default exports conditions to all internal packages`
